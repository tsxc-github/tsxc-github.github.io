墨迹触连

项目简介

一句话亮点
将拥有纸质感的电子墨水屏，变成一张会“联网”的智能卡片：内容远程下发永久显示，手机轻触（NFC）即刻打开对应应用或链接，磁吸设计让它随手可贴于你视野所及之处。🧲📱

它是什么？
墨迹触连是一款革命性的超低功耗电子墨水卡片设备。它通过云端下发文字、图片、二维码等内容到屏幕上，并能在​​静态显示模式下零功耗长久驻留​​；内置​​NFC​​功能，只需轻轻一碰，即可直达指定的网页链接、应用深层页面或自动保存 vCard 联系人信息。其磁吸外壳设计巧妙，可吸附于冰箱、显示器、白板或金属桌脚，完美融合​​视觉内容​​与​​触手可及的交互体验​​。

核心能力

  • 联网更新与模板展示： 一键更新「每日一言」、名片信息、明信片祝福、待办清单/公告等内容模板。
  • NFC 一碰直达： 瞬间打开链接/应用特定页面、自动保存 vCard。支持动态跳转与批量设备快速配置。
  • 纸质感墨水屏： 高对比度、无背光、类纸观感；静态显示零耗电，实现超长待机。
  • 磁吸即贴即用： 磁吸背壳便捷吸附，可更换前盖面板与个性化插画卡纸，并支持桌面底座形态。
  • 便捷设备管理： 多设备分组管理、远程内容刷新、固件升级与回滚。
  • 安全可控： 令牌访问校验、可控的分享链接时效性、可选内容离线展示保障隐私。

典型应用场景

  • 智能名片： 屏幕展示头像、姓名与动态二维码；手机轻触一键保存联系人信息或跳转官网/微洽。
  • 电子明信片： 传递节日祝福、旅行寄语；NFC一碰直达电子相册、祝福视频或地标定位。
  • 桌面信息看板： 展示每日一言、待办事项、会议室占用状态、计时提醒。
  • 商业展示助手： 商品价格牌/二维码引导、展品说明卡；轻触直达小程序、AR体验或下单页面。
  • 团队信息站： 轮值排班表、公司公告、Wi-Fi连接码，一键群发更新至所有设备。

设计亮点

  • 所见即所连： 纸质感呈现信息，指尖轻触完成连接，实物与数字世界无缝桥接。
  • 极致续航： 按需更新屏幕，深度休眠模式保障超低功耗，典型场景下可维持极长时间。
  • 轻量部署： 支持 USB-C 直连供电或电池供电（可选充电），磁吸随贴即用；手机轻松完成配网。
  • 开放可拓展： 支持模板与内容定制，开放 API，适合个人创意分享或小团队敏捷部署。

形态与规格（初稿）

  • 屏幕： 2.13英寸 / 2.9英寸 黑白或黑白红电子墨水屏，支持全局/局部刷新。
  • 连接： Wi-Fi / BLE 蓝牙配网，内容云端下发；NFC 支持 NDEF 标签（URL / 应用深层链接 / vCard）。
  • 供电： USB-C 供电或内置电池版本（可选充电管理），背部磁吸设计 + 铁氧体防磁隔离层。
  • 软件： Web端/小程序配置工具、内容模板库、设备管理后台与固件（OTA）升级系统。

名字释义

  • 墨迹： 电子墨水的纸质感呈现，如墨落纸面般沉静隽永。
  • 触连： NFC轻触瞬间的连接，打通物理媒介到数字世界的桥梁。

风格建议：

  • 当前版本： 兼顾技术与场景描述，适合放在官网产品页、介绍文档、部分演示PPT中。
  • 文艺版： 可大幅减少技术参数描述（尤其是规格部分），强化情感连接和故事性（如“把旅途的风景定格在卡片上，让远方的问候一触即达”），名字释义部分可以更加诗意化。适合众筹预热文案、社交媒体传播。
  • 硬核技术版： 需强调具体技术细节、协议支持（如NFC Type？ BLE版本？）、功耗参数（如静态功耗 uA 级？电池容量/待机时长估算？）、安全机制（加密方式？API规范？）。适合向极客、技术决策者或供应链伙伴介绍。
  • 商业版： 聚焦ROI（投资回报率）、具体行业应用价值（针对不同客户）、对比优势（相比传统纸质卡片/标牌/小屏幕）、商业模式（硬件+SaaS？按设备/功能收费？）、团队资质背书。适合给投资人、企业客户决策层看的简报。

应用场景

以下是针对「墨迹触连」设备的 15个创新应用场景，结合其核心功能（长久显示 + NFC 触达 + 磁吸便携 + 超低功耗），覆盖生活、商业与行业领域，突出差异化价值：

一、商业营销与客户触达

  1. 餐饮智能桌牌
    → ​​痛点​​：纸质菜单更新慢，促销信息难同步
    → ​​方案​​：桌面磁吸展示今日推荐菜+动态二维码，手机一碰跳转小程序点单/领优惠券
    → ​​价值​​：减少服务人力，实时更新菜品库存和折扣
  2. 零售电子价签PLUS版
    → ​​痛点​​：传统电子价签仅显价格，缺乏交互
    → ​​方案​​：商品旁磁吸展示价格+成分故事，NFC一碰直达溯源视频或AR试用
    → ​​价值​​：提升购物体验，减少纸质标签耗材
  3. 展会智能名片墙
    → ​​痛点​​：纸质名片易丢失，信息量有限
    → ​​方案​​：展台磁吸展示团队介绍与核心项目，访客手机轻触自动保存全员vCard
    → ​​价值​​：高效获客，强化科技感形象

二、效率与协同办公

  1. 工位智能状态牌
    → ​​痛点​​：同事频繁打扰“勿扰”状态
    → ​​方案​​：磁吸屏显示“会议中/专注/可沟通”，NFC碰触自动预约日历时段
    → ​​价值​​：无声传递状态，提升专注效率
  2. 会议室智能导引
    → ​​痛点​​：临时会议找不到空闲会议室
    → ​​方案​​：门外磁吸屏动态显示预约状态，NFC一碰查看实时占用日历/紧急申请
    → ​​价值​​：减少行政协调成本

三、生活与健康场景

  1. 家庭健康提醒站
    → ​​痛点​​：老人易忘记服药和体检
    → ​​方案​​：冰箱磁吸屏显示用药清单+提醒，子女远程更新日程；NFC碰触呼叫紧急联系人
    → ​​价值​​：远程关怀，避免健康风险
  2. 健身器械指导卡
    → ​​痛点​​：健身房新人不会使用器械
    → ​​方案​​：器械旁磁吸展示动作图解，NFC碰触播放教学视频/记录训练组次
    → ​​价值​​：替代纸质说明，提升健身体验

四、教育文化场景

  1. 博物馆文物故事卡
    → ​​痛点​​：展品文字说明枯燥，多语言支持难
    → ​​方案​​：展柜磁吸屏展示文物关键词，NFC一碰直达3D复原动画/多语言解说
    → ​​价值​​：沉浸式观展体验，减少导览器成本
  2. 学校课程公告板
    → ​​痛点​​：教室纸质课表更新滞后
    → ​​方案​​：门口磁吸屏显示今日课程+作业，教师远程更新；NFC碰触查看课件资源
    → ​​价值​​：实时同步信息，减少纸张浪费

五、特殊行业解决方案

  1. 医院电子床头牌
    → ​​痛点​​：纸质床头牌易污染，信息更新慢
    → ​​方案​​：磁吸显示患者用药/检查安排，护士NFC触碰确认执行并同步系统
    → ​​价值​​：降低院感风险，提升护理效率
  2. 工厂设备维保卡
    → ​​痛点​​：设备操作手册厚重，维护记录难追溯
    → ​​方案​​：机器旁磁吸展示操作要点，NFC碰触扫码登记维保记录/调取历史数据
    → ​​价值​​：无纸化流程，保障生产安全

六、创意互动玩法

  1. 线下解谜游戏线索卡
    → ​​痛点​​：实景游戏道具复用率低
    → ​​方案​​：场地内隐藏磁吸卡片显示谜面，玩家NFC碰触获取动态线索/解锁AR关卡
    → ​​价值​​：道具可远程重置，降低运营成本
  2. 婚礼电子请柬墙
    → ​​痛点​​：传统请柬信息固定，婚礼流程难同步
    → ​​方案​​：现场磁吸屏展示新人故事+座位表，宾客NFC碰触保存电子请柬/接收流程提醒
    → ​​价值​​：增强宾客参与感，节省印刷费用

七、隐藏场景：反常识但刚需

  1. 宠物防走失项圈牌
    → ​​痛点​​：传统狗牌信息量有限，联系失效无法更新
    → ​​方案​​：宠物项圈磁吸卡片显示“我是家养宠物”，拾遗者NFC碰触查看主人实时留言+宠物健康信息
    → ​​价值​​：动态更新联系方式，提升寻回率
  2. 家政服务验收卡
    → ​​痛点​​：保洁服务后无数字化反馈
    → ​​方案​​：服务结束磁吸屏显示清洁报告+评分二维码，户主NFC碰触付款/投诉
    → ​​价值​​：服务流程闭环，提升双方信任

场景落地支持建议：

  1. 为不同场景预置模板库（如餐饮菜单模板/医院床头牌模板）
  2. 定制行业版固件（例如医疗版增加紧急呼叫协议）
  3. 开发场景套装（含磁吸底座+防水壳+场景说明书）

这些场景均突出 “静态信息长效展示 + 零接触交互闭环” 的独特优势。如需进一步深化某个领域（如零售/医疗），我可提供具体场景的 技术配置方案 + 成本测算 ✨

时间线

推荐整体架构

  • 设备端：ESP32（LILYGO T5 2.13"）+ NTAG I2C Plus（I2C）+ ePaper
  • 连接方式：设备保持到云端的长连接（MQTT/MQTTS 或 WebSocket）。云端“主动推送”更新，设备实时处理
    • 说明：直接给设备“开放入站端口”在公网/移动网络/NAT 下几乎不可行；用 MQTT/WebSocket 的持久出站连接是业界通用的云端主动下发方式
  • 推送内容：{ nfc_url, display_payload }，设备收到后写 NDEF 并刷新屏

采购清单（新增/重点）

  • NTAG I2C Plus 小板 ×2（NT3H2211/NT3H1101 任一；就近采购谁快用谁）
  • LILYGO T5 2.13" ×2（带屏）
  • 电池 3.7V 500 mAh（带保护）×3；JST-PH2.0 线若干
  • 磁铁 N52 8×2 mm ×8–12；0.3–0.5 mm 铁氧体片 ×1 张
  • M2 热熔铜柱/螺丝、0.5–1 mm 泡棉、PLA 线材、侧滑电源开关

注意事项

硬件连接（T5 → NTAG I2C Plus）

  • 电源：3V3、GND
  • I2C：SDA → GPIO21，SCL → GPIO22（T5 常用；以丝印/原理图为准）
  • 中断/GPO：不接（你已确认不需要近场唤醒）
  • 注意：I2C 需上拉（常见小板自带）；线尽量短，远离磁铁

外壳条屏版（Rev.A）要点

  • 外形参考：长 110–120 mm × 宽 38–45 mm × 厚 10–12 mm
  • 结构：纯平面前框 + 背壳，两片式；USB‑C 开口、侧边电源滑开关
  • NFC 与磁吸：NFC 线圈贴近前面板中心偏一侧，背面垫铁氧体；四角内嵌磁铁并与线圈保持 ≥3–5 mm 间距
  • 装配：主板+屏限位，电池独立腔 + 泡棉限位；FPC 转折半径 ≥1 mm

云端与消息约定

  • 连接：MQTTS（推荐），设备 ClientID = device_id，使用令牌/证书鉴权
  • 订阅主题：mxjl/{device_id}/cmd
  • 上报主题：mxjl/{device_id}/ack
  • 消息示例（cmd）：{ “nfc_url”: “https://your.link/abc”, “display”: { “type”: “image_url”, “url”: “https://cdn/xxx.avif” }, “ts”: 1721234567 }
  • 设备逻辑：
    • 收到 cmd → 若 nfc_url 变更，写 NDEF（URI 记录）；若 display 变更，拉取位图并全刷
    • 成功后 ack：{ “ok”: true, “applied”: [“nfc”,“display”], “fw”:“0.1.0” }

固件实现提示

  • NTAG I2C 库：优先使用现成 Arduino 驱动（NTAG I2C Plus/NT3H2211 相关库）+ NDEF 编解码
  • 低功耗：为满足“云端主动下发”，MVP 采用“持续在线 + Modem-sleep”，待机电流会高于纯深睡
    • 体验预估：在线空闲 10–20 mA（随板卡而异）→ 500 mAh 约 1–2 天；演示无压力，后续再做“唤醒窗口”优化
    • 长期闲置可用物理电源开关彻底断电

读距与体验

  • 典型读距 2–4 cm（看外壳厚度/手机而定）；前面板尽量薄且非金属
  • URL 动态更新即时生效，手机下一次碰即可打开最新链接
